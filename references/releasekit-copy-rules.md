# ReleaseKit Copy Rules for AI and Privacy

Use these rules for App Store metadata, release notes, onboarding, privacy pages, settings text, and review notes.

## Voice

Use clear product language. Avoid engineering terms.

Avoid user-facing terms:

- prompt
- system prompt
- agent pipeline
- LLM context
- chain of thought
- internal instruction
- hidden rule
- vector store

Preferred terms:

- AI suggestion
- draft
- selected content
- local processing
- privacy control
- review before saving
- external AI

## Required Statements When External AI Exists

Include concise copy that communicates:

- AI features are optional if the product supports disabling them.
- Selected content may be processed to generate suggestions.
- Suggestions should be reviewed by the user.
- The app does not silently send all content.
- The user can manage AI/privacy settings.

## App Store Description Pattern

Use restrained claims:

"AI can help organize selected content into draft summaries, categories, and next steps. You remain in control of what is saved, exported, or shared."

## Privacy Copy Pattern

"When enabled, external AI features process only the content you choose for that feature. The app is designed to keep core records usable without external AI."

## Settings Copy Pattern

- Title: "AI Suggestions"
- Toggle: "Allow external AI drafts"
- Helper: "When enabled, selected content may be used to create draft suggestions. Review suggestions before saving or sharing."

## Release Notes Pattern

"Adds privacy controls for AI draft suggestions and keeps core workflows available when external AI is off."
